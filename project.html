Winery

동기: final-project의 주제 선정은 자유였지만, 현재 유사한 사이트가 드물고 다른 조와 차별점을 줄 수 있는 부분에 초점을 두고
  회의를 진행했습니다. 와인을 구매할 때 해외 'Vivino'라는 앱을 이용 중이었는데 'Vivino'의 장점과 개인 취향대로 구독도 할 수 있고
  배송 시스템까지 도입된 평소 원하던 사이트의 의견을 제안했고, 이를 토대로 진행하게 되었습니다.

Basic: Spring MVC 패턴으로 흐름을 두고, MyBatis의 xml방식과 Annotation방식을 사용하여 CRUD를 동작했고,
   CSS부분은 수정 시 발생할 수 있는 충돌을 방지하기 위해 각각 파일을 생성했습니다.

MY PART

IntroPage.

@SessionAttributes("alias")와 @ModelAttribute("alias")로
동기화가 적용되게하고, 단일파일업로드를 추가해 img까지
추후 부분 수정이 가능하게 하였습니다.
값은 JSTL을 이용하여 해당 영역에 묘출되게 하고,
기본 디자인 틀은 Bootstrap을 이용하였습니다.
하단에는 jQuery fadeIn/Out 함수를 사용해
Top버튼을 생성하고 클릭 시 상단으로 이동할 수 있도록 했습니다.

NoticePage.

게시판을 위한 테이블을 하나로 이용하기 위해
각각 구분번호로 분류하는 column을 추가하였고,
공통적으로 CRUD와 paging, 조회 수를 구현했습니다.
textarea영역에 내용 차이로 인한 높이의 변동을 jQuery를 사용하여
height를 auto로 설정, 기본 속성인 overflow를 hidden 처리하고
resize를 none으로 하여 사용자가 임의로 사이즈 변경을 방지했습니다.

FaqPage.

@SessionAttributes("alias")와 model.addAttribute("alias")를
공지사항, FAQ, QnA 모두 수정이 가능하게 동기화를 적용했습니다.
사용자의 편의를 위해 JSTL Core Tag와 input의 value를 비교하여
소분류 필터기능을 구현했습니다.
JavaScript toggle을 사용하여 Accordion방식으로 표현하고
textarea영역은 위와 동일하게 적용해 주었습니다.       

QnaPage.

HttpSession을 이용하여 작성자의 value는
session.getAttribute("userID").toString()으로 가져와서
readonly 속성을 적용해 임의로 변경할 수 없도록 하고,
JavaScript를 사용해 location.href에 해당 번호를 전달해서
상세보기 페이지로 이동할 수 있게 연결했습니다.
두 테이블을 LEFT OUTER JOIN하여 답변대기/답변완료 필터기능을 구현했습니다.
